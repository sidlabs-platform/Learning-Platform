## User Story

**As a** developer,
**I want** all ten domain entities defined as SQLAlchemy ORM models with proper constraints, relationships, and an Alembic migration that creates the schema on first run,
**so that** every service can persist data correctly from the first day of development.

## Acceptance Criteria

1. **Given** the Alembic migration `001_initial_schema` is run,
   **When** the database is inspected,
   **Then** all 10 tables exist with the correct columns, FK constraints, UNIQUE constraints, and CHECK constraints.

2. **Given** a `Course` ORM object is saved with `learning_objectives=["obj1", "obj2"]`,
   **When** retrieved from the database,
   **Then** `learning_objectives` is deserialized back to `["obj1", "obj2"]` (JSON TypeDecorator round-trip).

3. **Given** a `Course` is deleted,
   **When** the database is queried for its `Modules`, `Lessons`, `QuizQuestions`, and `Enrollments`,
   **Then** all are also deleted (CASCADE delete).

4. **Given** a `User` ORM object is created with `role="superadmin"`,
   **When** the database attempts to save it,
   **Then** a CHECK constraint violation error is raised.

## Technical Notes

- **Stack:** SQLAlchemy 2.x async, aiosqlite, Alembic 1.x
- **Key considerations:** Use `JSONList` TypeDecorator for `list[str]` fields (`learning_objectives`, `tags`, `options`); all PKs are UUID v4 strings generated by `_uuid()`; use `mapped_column` with `Mapped[]` type annotations (SQLAlchemy 2.x style)
- **Configuration:** `DATABASE_URL` env var (default: `sqlite+aiosqlite:///./learning_platform.db`)
