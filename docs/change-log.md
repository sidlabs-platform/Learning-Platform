Waves 3–6: service layer build-up (auth → course → progress → AI → reporting). Wave 7: routers + first frontend templates. Wave 8: tests + remaining frontend. Full BRD traceability: every task maps to a story → epic → BRD requirement IDs. Key architectural decisions reflected in task designs: 4-layer AI generation module (Prompt Orchestration → Model Invocation → Content Persistence → Review Workflow), upsert-based progress recording for refresh safety, server-side quiz evaluation, streaming CSV export, bleach sanitisation at write-time + render-time, JWT in HTTP-only SameSite=Strict cookies. |
